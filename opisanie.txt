1. Пользователь с помощью клиентского приложение может настраивать IP-адрес и номер порта, используемого сервером.
2. У каждого клиента на сервере есть свой каталог, в который он может загружать свои файлы, удалять, перименовывать, создавать новые поддиректории.
3. Доступ к каталогам других пользователей ограничен, файлы можно только скачивать, удалять и загружать свои нельзя.
4. Сревер обеспечивает процедуру проверки подлинности с помощью логина и пароля.
5. Количество возможных загрузок ограничено (например, в сутки), для того, чтобы этип не могли воспользоваться злоумышленники и не засоряли сервер огромным числом файлов.
Также существует ограничение по объему загружаемого файла.
6.Команды, которые может использовать пользователь:
-server help - выдача системы команд и пояснения к ним
-server ch_dir <путь> - сменить директорию на сервере
-server mk_dir <name> - создать директорию
-server download <имя файла> - загрузить файл с сервера (из каталога, в котором сейчас 					находится пользователь на сервере)
-server upload <имя файла> - загрузить файл на сервер(из каталога, в котором сейчас 					находится пользователь на ПК)
-server view - посмотреть дерево каталогов на сервере
-server connect <IP> <номер порта> - подключиться к серверу с укзаанием IP и номера порта
-server disconnect - разорвать текущее соединени с сервером
